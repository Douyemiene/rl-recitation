Core Definitions

The Value Function · State-Value \(V\) · Action-Value \(Q\)

Section 1 · Core Definitions

A reward measures the current step; a value function measures the entire future

Reward signal
$$r_t$$

The reward signal captures immediate feedback only. It provides no information about future consequences of the current action.

Discounted total reward (class slide 19)
$$R(\infty) = \sum_{t=0}^{\infty} \mathbb{E}\!\left[\gamma^t r(a(t), s(t))\right]$$

The discounted total reward accumulates all future rewards, weighted by \(\gamma^t\). The value function is the expected value of this quantity from a given state.

Because the optimal policy maximises total future reward rather than immediate reward, the value function is the correct objective for agent decision-making.

Section 1 · Core Definitions

The state-value function \(V_\pi(s)\) gives the expected return from each state under policy \(\pi\)

Definition (class slide 28)
$$V_\pi(s) = \mathbb{E}_{a \sim \pi(s),\, P,\, r}\!\left[\sum_{t=0}^{\infty} \gamma^t r\!\left(a(t),s(t)\right) \;\middle|\; s(0) = s\right]$$

The expectation is taken over the stochastic policy, the transition dynamics, and the reward function.

Due to the Markov property, the optimal policy at any time is a function of the current state alone.

Gridworld analogy

A cell located one step from the goal carries high value regardless of the path taken to reach it.

What \(V_\pi(s)\) does not provide

The state-value function evaluates positions. It does not specify which action to take. That is the role of \(Q_\pi(s,a)\).

Section 1 · Core Definitions

The action-value function \(Q_\pi(a,s)\) evaluates a specific action taken at a given state

Definition (class slide 30)
$$Q_\pi(a,s) = \mathbb{E}\!\left[r(a,s) + \sum_{t=1}^{\infty} \gamma^t r\!\left(\pi(s(t)),s(t)\right) \;\middle|\; s(0)=s,\; a(0)=a\right]$$

The agent commits to action \(a\) at the current step and follows policy \(\pi\) from the next state onward.

Why Q is critical for action selection

\(Q_\pi(a,s)\) is critical for choosing between different actions in the same state. Because the action is built into the input, the agent does not need to know the transition model \(P(s'|s,a)\) to select the best move.

Relationship to \(V_\pi\)
$$V_\pi(s) = \sum_a \pi(a|s)\; Q_\pi(a,s)$$

The state value equals the expected action value under \(\pi\).

Section 1 · Core Definitions

Both forms are correct and apply in different situations

Form 1 · Policy evaluation
$$V_\pi(s) = \sum_a \pi(a|s)\; Q_\pi(a,s)$$

When to use it: the agent follows a stochastic policy, such as the 50/50 example.

The value of the state is the average of all action values, weighted by how likely the agent is to take each action.

Restaurant analogy

You pick a dish at random. The value of your meal is the average quality across everything on the menu, weighted by how often you order each dish.

Form 2 · Optimal control
$$V^*(s) = \max_a\; Q^*(a,s)$$

When to use it: the agent follows the optimal policy \(\pi^*\).

The value of the state is the value of the single best action available. All suboptimal actions are ignored.

Restaurant analogy

You know exactly which dish is best. The value of your meal is the quality of that one dish alone — the rest of the menu is irrelevant.

Policy Fundamentals

What is a Policy · Stochastic · Deterministic vs Stochastic

Section 2 · Policy Fundamentals

A policy \(\pi\) is the agent's decision rule: a mapping from every state to an action or a distribution over actions

Deterministic policy (class slide 19)
$$\pi: S \to A \qquad \pi(s) = a$$

The policy assigns a single action to each state with certainty. There is no randomness in the selection.

Stochastic policy (class slide 19)
$$\pi(a \mid s) = P(A_t = a \mid S_t = s)$$

The policy specifies a probability distribution over actions given state \(s\). The agent samples from this distribution to select the next action.

Goal (class slide 29)

Every RL algorithm searches for the optimal policy \(\pi^*\) that maximises the state-value function \(V_\pi(s)\) for all states. Value functions are instruments for evaluating and improving the policy.

$$\pi^* = \arg\max_\pi V_\pi(s) \quad \forall s \in S$$
Section 2 · Policy Fundamentals

A stochastic policy assigns a probability distribution over actions rather than a single fixed choice

Example: uniform exploration policy
$$\pi(a \mid s) = \begin{cases} 0.5 & \text{if } a = 1 \\ 0.5 & \text{if } a = 2 \end{cases}$$

This represents a uniform random strategy, applied when the agent has not yet determined which action yields a higher return.

Implied state-value
$$V_\pi(s) = 0.5 \cdot Q_\pi(1,s)\;+\;0.5 \cdot Q_\pi(2,s)$$

As the agent accumulates experience, the policy shifts weight toward the action with the higher Q-value.

Why begin with a stochastic policy?

An agent that always selects the same action cannot discover that an alternative action yields a higher return. Stochastic behaviour drives exploration of the state space.

Why converge to a deterministic policy?

For tabular MDPs, there always exists a deterministic optimal policy (class slide 31). Once \(Q^*\) is known, randomness reduces performance.

The Bellman Equation

Recursive Decomposition · MDP Forms · Discount Factor \(\gamma\)

Section 3 · The Bellman Equation

The value function satisfies a recursive decomposition into immediate reward and discounted future value

Derivation (class slide 34)
$$V(s) = \mathbb{E}\!\left[r_0 + \gamma r_1 + \gamma^2 r_2 + \cdots \;\middle|\; s_0 = s\right]$$
$$= \mathbb{E}\!\left[r(s)\right] + \gamma \,\mathbb{E}_{s' \sim P(\cdot|s_0=s)}\!\left[V(s')\right]$$

The value decomposes into two components: the expected immediate reward \(\mathbb{E}[r(s)]\) and the discounted expected value of the successor state \(\gamma\,\mathbb{E}[V(s')]\).

Matrix form (class slide 35)
$$\mathbf{V} = \mathbf{r} + \gamma \mathbf{P}\mathbf{V}$$

This yields a linear system solvable by matrix inversion:

$$\mathbf{V} = (I - \gamma \mathbf{P})^{-1}\mathbf{r}$$

Computational complexity is \(O(n^3)\) for \(n\) states.

Section 3 · The Bellman Equation

The Bellman equations for an MDP relate \(V_\pi\) and \(Q_\pi\) through the policy and transition dynamics

Bellman equation for \(V_\pi\) (class slide 38)
$$V_\pi(s) = \sum_{a \in \mathcal{A}} \pi(a|s)\; Q_\pi(a,s)$$

The state value is the expectation of the action value under the current policy.

Bellman equation for \(Q_\pi\) (class slide 38)
$$Q_\pi(a,s) = \mathbb{E}\!\left[r(a,s)\right] + \gamma \sum_{s' \in \mathcal{S}} P(s'|s,a)\,V_\pi(s')$$

The action value equals the immediate reward plus the discounted value of the resulting state.

Combined form
$$V_\pi(s) = \mathbb{E}\!\left[r(a,s)\right] + \gamma \sum_a \pi(a|s) \sum_{s'} P(s'|s,a)\,V_\pi(s')$$

Substituting the \(Q_\pi\) equation into the \(V_\pi\) equation gives the full Bellman expectation equation for an MDP.

Section 3 · The Bellman Equation

\(\gamma \in [0,1]\) sets how quickly the weight of a reward decays with time

$$R(\infty) = \sum_{t=0}^{\infty} \gamma^t\, \mathbb{E}\!\left[r(a(t),s(t))\right]$$

The weight of a reward at step \(t\) is \(\gamma^t\). As \(t\) grows, this weight shrinks — faster for small \(\gamma\), slowly for \(\gamma\) near 1.

Optimal Decision Making

Bellman Optimality · Greedification · Value and Policy Iteration

Section 4 · Optimal Decision Making

Bellman optimality replaces the policy expectation with a maximisation over actions

Bellman expectation (any \(\pi\))
$$V_\pi(s) = \sum_a \pi(a|s)\sum_{s'} P(s'|s,a)\!\left[r + \gamma V_\pi(s')\right]$$
Bellman optimality (class slide 31)
$$V^*(s) = \max_a \sum_{s'} P(s'|s,a)\!\left[r + \gamma V^*(s')\right]$$
Greedification of \(Q^*\) yields \(\pi^*\) (class slide 31)
$$\pi^*(s) = \arg\max_a\; Q^*(a,s) \qquad V^*(s) = V_{\pi^*}(s)$$

For tabular MDPs, a deterministic optimal policy always exists. Selecting the highest \(Q^*\) value at each state is sufficient for global optimality.

Why \(Q^*\) is easier to act on than \(V^*\)

Maximising \(V^*(s)\) requires knowing the transition model \(P(s'|s,a)\) to determine which action leads to which next state.

With \(Q^*\), the action is already built into the function. The agent does not need to know the environment's dynamics and simply selects:

$$\pi^*(s) = \arg\max_a\; Q^*(a,s)$$
Section 4 · Optimal Decision Making

A single greedy step with respect to \(V^*\) is globally optimal because \(V^*\) encodes the complete future

Why one step is sufficient

\(V^*(s')\) contains the optimal expected return for every state reachable from \(s'\). By recursive definition it encodes all future rewards to the horizon. The agent therefore needs to evaluate only the immediate step.

$$\pi^*(s) = \arg\max_a \!\left[\,r(a,s) + \gamma\sum_{s'} P(s'|s,a)\,V^*(s')\right]$$
Policy search (class slide 32)

It is sufficient to learn either the optimal policy \(\pi^*\) directly (policy-based) or the optimal action-value function \(Q^*\) (value-based). When the MDP is known, these are addressed by policy iteration and value iteration.

Section 4 · Optimal Decision Making

Value iteration and policy iteration both converge to \(\pi^*\) by applying the Bellman operator iteratively

Value Iteration (class slide 32)
$$V_{k+1}(s) = \max_a\!\left[r(a,s) + \gamma \sum_{s'} P(s'|s,a)\,V_k(s')\right]$$

All states are swept repeatedly using the Bellman optimality operator as the update rule. The procedure repeats until \(\|V_{k+1}-V_k\|_\infty < \varepsilon\).

Straightforward to implement; one sweep per iteration.

Policy Iteration (class slide 32)

1. Evaluate: solve \(V^\pi\) exactly for the current policy \(\pi\).

$$\pi'(s) = \arg\max_a\!\left[r(a,s) + \gamma\sum_{s'} P(s'|s,a)\,V^\pi(s')\right]$$

2. Improve: set \(\pi \leftarrow \pi'\) and repeat until \(\pi\) is stable.

Fewer outer iterations; each evaluation step is more costly.

Both converge to the unique \(V^*\) by the Banach fixed-point theorem.

Summary

Four ideas that constitute value-based reinforcement learning

1 · Core Definitions

\(V_\pi(s)\) estimates how desirable a state is. \(Q_\pi(a,s)\) evaluates specific actions and is critical for choosing between them without a transition model.

2 · Policies

\(\pi\) maps states to actions or distributions. Stochastic policies explore; deterministic policies exploit. The goal is to find \(\pi^*\) that maximises \(V_\pi(s)\) for all \(s\).

3 · Bellman

The value function satisfies \(V(s) = \mathbb{E}[r] + \gamma\,\mathbb{E}[V(s')]\). In matrix form: \(\mathbf{V} = \mathbf{r} + \gamma\mathbf{P}\mathbf{V}\). The discount \(\gamma\) controls planning horizon.

4 · Optimality

Greedification of \(Q^*\) yields \(\pi^*\) without requiring a transition model. Value and policy iteration both converge to the unique \(V^*\).
